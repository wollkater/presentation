Thin vs Zero Clients

Thin Clients

Minimale Hardware, Software auf dem Client

Betriebssystem auf dem Client installiert

Software kann auch installiert werden

Rechenintensive Operationen werden auf den Server ausgelagert

Vorteile

  • Eigene Leistung

Nachteile

  • Betriebssysteme müssen noch lokal gemanaget werden (Updates, etc.)

Zero Clients

Ähnlich wie Thin Clients, speichern aber keine Daten lokal

Betriebssystem und Software liegen auf dem Server

Vorteile

  • Betriebssystem kann zentral gemanaget werden
  • schnellere Bootzeiten
  • Sicherer

Nachteile

  • Keine lokalen Speichermöglichkeiten
  • Herstellerspezifisches Protokoll (Wechsel wird erschwert)
